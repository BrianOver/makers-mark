# 3D Town Hub + Interaction — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the 2D floating-sprite town hub with a grounded true-3D low-poly town (generic Kenney CC0 meshes) where the player avatar walks (WASD + click-to-move), buildings read as real structures, interaction is obvious (proximity highlight + prompt → existing panels), and the menu-sizing defects are fixed.

**Architecture:** A `Node3D` world lives inside a `SubViewport` (via `SubViewportContainer`, `Stretch=true`) as the base child of the existing `main_ui.tscn` Control tree; the 2D HUD/`DrawerHost` draw on top unchanged. The world reads `SimAdapter.CurrentState` and re-emits the existing `BuildingClicked`/`HeroClicked` string vocabulary into `MainUi` — zero sim change (KTD2). Real 3D depth replaces Y-sort. `NavigationAgent3D` (baked from building footprint colliders) drives click-to-move; nav + positions are presentation-only and never feed the sim.

**Tech Stack:** Godot 4.6.3-stable .NET (C#, `net10.0` in `godot/GodotClient.csproj`), gdUnit4Net (`gdUnit4.api 5.0.0`), Kenney CC0 GLB kits via Git LFS.

## Global Constraints

- Engine: **Godot 4.6.3-stable .NET only** (`.godot-version` is source of truth). Never open/re-save `godot/` with any other editor.
- `godot/GodotClient.csproj` keeps `net10.0` explicitly; never add a TFM elsewhere; never commit `net8.0`.
- **Sim purity (KTD2):** `sim/GameSim/` gets ZERO changes. No RNG, no wall-clock, no `Math.*` transcendentals in any code that could reach the sim. Nav/positions/deltas are presentation-only and must never be read back into a sim action beyond discrete intents (e.g. "player interacted with Forge").
- **Determinism:** same seed + same actions = identical sim state. This plan touches only `godot/` presentation; the golden-replay/fast-lane suites must stay green untouched.
- **Deny-list — never edit:** `Game.sln`, `godot/project.godot`, `.github/`, `sim/GameSim/Contracts/`, `CLAUDE.md`, `global.json`, `Directory.Build.props`, `.godot-version`. Input actions stay **runtime-registered** (as today in `WorldInput`) so `project.godot` is never touched. Menu clipping is fixed in layout, not by a `[display]` window size.
- **Tests:** gdUnit4Net files wrapped in `#if GDUNIT_TESTS ... #endif`, `[TestSuite]` + `[RequireGodotRuntime]`, **scenes built in code (no `.tscn` authoring)**, real `SimAdapter` (`GameFactory.NewGame(seed) with {...}`), `try/finally` `Free()`/`Unmount`, capped loops, `ToSignal` frame pumps from `UiTestSupport`. **Never rely on headless physics *picking*** — proven dead on this build (G1 verdict); use the world-space fallback that emits the real signal.
```csharp
using Godot;
namespace GodotClient.Town3d;

public partial class CameraRig : Node3D
{
    [Export] public Node3D? Target;
    [Export] public float Pitch = -50f;
    [Export] public float Distance = 14f;
    [Export] public float FollowSpeed = 5f;
    private Camera3D _cam = null!;

    public override void _Ready()
    {
        _cam = new Camera3D { Name = "Camera3D", Fov = 45f, Near = 0.5f, Far = 200f };
        AddChild(_cam);
        _cam.RotationDegrees = new Vector3(Pitch, 0, 0);
        _cam.Position = _cam.Basis.Z * Distance;   // back along its own +Z, looking at rig origin
        _cam.Current = true;
        if (Target != null) GlobalPosition = Target.GlobalPosition;
    }

    public override void _Process(double delta)
    {
        if (Target == null) return;
        float t = 1f - Mathf.Exp(-FollowSpeed * (float)delta);
        GlobalPosition = GlobalPosition.Lerp(Target.GlobalPosition, t);
    }
}
```

- [ ] **Step 3: Implement PlayerController.cs** (WASD camera-relative; movement guidance from research §1)

```csharp
using Godot;
namespace GodotClient.Town3d;

public partial class PlayerController : CharacterBody3D
{
    [Export] public float Speed = 5f;
    [Export] public float TurnSpeed = 12f;
    public Camera3D Cam = null!;      // set by Town3D from the rig
    public Node3D Mesh = null!;       // visual child
    public bool IsClickMoving { get; protected set; }   // T6 sets true

    public override void _PhysicsProcess(double delta)
    {
        if (IsClickMoving) return;    // T6 handles nav follow; WASD path here
        ApplyWasd(delta);
    }

    public void ApplyWasd(double delta)
    {
        Vector2 input = Input.GetVector("move_left", "move_right", "move_up", "move_down");
        Vector3 dir = Cam.GlobalBasis * new Vector3(input.X, 0, input.Y);
        dir.Y = 0;
        dir = dir.LengthSquared() > 0 ? dir.Normalized() * input.Length() : Vector3.Zero;
        Velocity = new Vector3(dir.X * Speed, 0, dir.Z * Speed);
        MoveAndSlide();
        Face(dir, (float)delta);
    }

    protected void Face(Vector3 dir, float dt)
    {
        if (dir.LengthSquared() < 0.0001f) return;
        float target = Mathf.Atan2(-dir.X, -dir.Z);
        Mesh.Rotation = new Vector3(0, Mathf.LerpAngle(Mesh.Rotation.Y, target, TurnSpeed * dt), 0);
    }
}
```
